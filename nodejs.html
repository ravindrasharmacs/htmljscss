Question:

Q1. How does event loop architecture work in Node.js?

Event Loop Phases:

Timers: This phase executes callbacks scheduled by setTimeout() and setInterval(). If there are any timer callbacks that are due to execute, they will be processed here.

I/O Callbacks: Executes I/O callbacks deferred from the previous cycle. These include almost all callbacks except for close callbacks, the ones scheduled by timers, and setImmediate.

Idle, Prepare: Only used internally in Node.js for specific operations.

Poll: This phase retrieves new I/O events and executes their callbacks. During this phase, the event loop processes events for an indefinite period until the polling queue becomes empty.

Check: Executes callbacks scheduled by setImmediate(). These callbacks are executed immediately after the polling phase.

Close Callbacks: Executes close callbacks, such as socket.on('close', ...).

How it Works:

When Node.js starts, it initializes the event loop, processes the input script, and then starts executing the event loop.

Asynchronous operations, such as I/O and timers, are scheduled in the corresponding phases of the event loop.

The event loop picks up these tasks in each cycle and processes them until there are no more tasks to handle.
